数位dp记忆化搜索

发表于2022-12-25|更新于2022-12-25|ACM动态规划数位dp
|阅读量:

题目描述

科协里最近很流行数字游戏。

某人命名了一种不降数，这种数字必须满足从左到右各位数字呈非下降关系，如123，446123，446123，446。

现在大家决定玩一个游戏，指定一个整数闭区间 [a,b]，问这个区间内有多少个不降数。

输入样例

输入包含多组测试数据。

每组数据占一行，包含两个整数aaa和bbb。

输出样例

每行给出一组测试数据的答案，即[a,b][a,b][a,b]之间有多少不降数。

记忆化搜索+数位dp

记忆化搜索模板其实是将原来循环迭代的数位dp解法的 动态规划 与 枚举数位过程结合在一起。

代码的解释

变量含义的解释

  • nums[] : 里面存储的是n的每一位数字，nums[1] 存的是 个位，nums[2] 是 十位,…
  • len : 表示 n 有 几位数.
  • f[i][j] : 满足当前数有i位，且最高位的数字为j的数字的个数
  • dfs(int len, int last, bool limit) : 还有len个位置需要填，len + 1 位置填了last，且此时状态为limit 时 填数的方案数
  • limit :
    • limit == 1 则当前可填的数字必须处于[0,nums[len]]的区间
    • limit == 0 则当前可填的数字无限制，处于[0,9]的区间
    • 比如说要我们枚举 1 - n 的数字，如果n = 123
      • _ _ _ 刚填数时，limit == 1，因为第一个空只能填 0，1 ，否则会大于n
      • 0 _ _ 假如第一个填的是0，limit == 0，因为第二个空我们能填 0-9
      • 实现limit的转化是代码中 limit && i == up ，可以先思考一下，等下详细说明(●’◡’●)
  • up : 当前位置我们能填的数字的上限
    • limit == 1 时，我们能填的数最大不能超过 n 当前位置上的数字,也就是 up == nums[len]
    • limit == 0 时，填的数字无限制 up == 9
  • ans : 当前状态下方案数

函数中一些语句的解释

  • if (!len) return 1 : len == 0 意味着数字已经填完，说明这是一种合法方案，所以返回1
  • if (!limit && f[len + 1][last] != -1) return f[len + 1][last]; : 这是记忆化搜索的主要函数，必须是 limit == 0 时才能返回 f[len + 1][last],这是因为我们的f表示的是在填数字是可以从 0-9 中任意挑选时的方案数，所以首先要 limit == 0
  • ans += dfs(len - 1, i, limit && i == up) : 转移方程
  • limit && i == up : 一些解释
    • 如果在枚举之前的数位时 limit == 0 那么后面的数位就可以 随便填 0 - 9的数字
    • 如果上一次 limit == 1 且当前 i == up(表示这次填的也是n数位上某个数字) ，说明下一位数填的时候还是存在限制.
  • return limit ? ans : f[len + 1][last] = ans : 这里同函数语句解释的第二条，只要当limit == 0 时我们才做记忆化搜索过程，唯有f[][]表示的状态是在没有limit限制下,才能保证它里面一直存储的是正确的值
